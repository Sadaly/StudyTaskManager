Примерная структура проекта																																					  
Предполагаю что будет 1 файл для 1 теста
  /Unit
    /Commands         # Тесты команд

                      # мб эти две папки лишние и будем определять все в файле самого теста
    /ResultsErrors    # Тесты кастомных результатов и ошибок
    /TestDoubles      # Тестовые реализации интерфейсов (Stubs/Mocks)
                      
  /Integration
    /Database         # Тесты с реальной DB (если нужно)
    /ExternalApi      # Тесты клиентов API
